# Phase 17 — the runner: completion mode + budget circuit breaker (§8 Phase 3; P0.2–4,7,8,9,10; P1.9; FP-3)

Deliver the `subscription-claude` backend as **completion mode** (one headless `claude -p` per reasoning call) + the circuit breaker that replaces the $0.50 loop brake, + zombie defense, health, quota throttle, and `get_runner_status`. **Agent mode (loopback MCP connect-back) is phase-19 — NOT this phase.** After this, the ProviderRouter (phase-16) can route the subscribable roles to the subscription when the user opts in; the circuit-breaker ships here, **before any scheduled use**.

**Prime directive:** ADD, never REMOVE. Ships OFF by default (`runner.enabled=false`) → a default install NEVER spawns `claude` and behaves identically. Every completion falls back (subscription→cloud-api→local) if the runner is unhealthy/disabled.

## `src/main/runner/` (new module)
- **`spawn.ts`** — `spawnClaude(argv, { stdin, cwd, timeoutMs, signal })`: argv-array spawn (NEVER `shell:true`), prompt on **stdin**, stdout+stderr captured, watchdog kills the **process tree** at timeout (`taskkill /T /F` win32; negative-pid group kill POSIX — spawn `detached` on POSIX for the group). Wrap every spawn in `telemetry.withSpan('runner.spawn', { 'runner.mode','runner.model','runner.task_id','runner.exit_code','runner.num_turns' })`. Record a `runner_runs` row: **`started_at` MUST be `new Date().toISOString()`** (ISO-8601 UTC — the phase-14 `CallBudget.windowUsage` contract), `pid`, `stderr_tail` (last ~2KB), `model`, `duration_ms`, tokens, `shadow_cost_usd` (= envelope `total_cost_usd`, labelled estimate), `is_error`, `exit_code`.
- **Envelope parsing** (defensive, §6.7/§10.12): parse `--output-format json` → `{ session_id, is_error, num_turns, duration_ms, usage, total_cost_usd, result }`. **Missing a required field** (`is_error`/`result`/`session_id`) ⇒ treat as a `not-installed`-grade failure, never crash the caller. Unknown fields ignored.
- **`binary.ts`** (P1.9/§10.12) — resolution order: `settings.runner.binaryPath` → probe `~/.local/bin/claude`, `%USERPROFILE%\.local\bin\claude.exe`, npm global prefix, `/usr/local/bin`, `/opt/homebrew/bin` → PATH. Windows `.cmd` shim: prefer native `claude.exe`; else `cmd.exe /d /s /c claude.cmd …` (args via the spawn array — NEVER `shell:true`). `RUNNER_MIN_CLI_VERSION` gate (`claude --version`). Surface the resolved absolute path in `get_runner_status`. **TEST SEAM (critical):** an env override `AGENTIC_OS_RUNNER_BINARY` (and `settings.runner.binaryPath`) wins over all probing; when it points at a `.mjs`/`.js`, spawn `process.execPath` + `[scriptPath, ...argv]` so a Node fake runs cross-platform with no shell.
- **`health.ts`** (P0.7/§9.7) — `RunnerHealth` cache (`RUNNER_HEALTH_TTL_MS` 15min): resolved binary + version + last state `'ok'|'not-installed'|'auth-expired'|'quota-exhausted'|'unknown'`. `runnerHealthy()` = enabled ∧ resolved ∧ state∈{ok,unknown} within TTL → **injected into the ProviderRouter as `runnerHealthy`**. `classifyRunnerFailure(envelope, stderr)` (ONE classifier): auth `/please run \/login|oauth|credential|authentication_error|invalid api key/i`; quota `/usage limit|rate limit|resets at|overloaded|429/i` (parse a reset time when present); else not-installed/envelope-drift.
- **`lanes.ts`** (P0.9/§9.8) — `runnerLiveLane` + `runnerBackgroundLane` (each concurrency 1); the background lane checks a "live busy" flag before each spawn (cooperative). `RUNNER_LANES`/`RUNNER_COMPLETION_CONCURRENCY` already in config.
- **`completion.ts`** — the completion-mode fn = the **`subscriptionComplete` injected into the ProviderRouter**. Per call: consult `CallBudget.checkBudget(taskId)` (P0.2, throws `CallBudgetExceededError extends SpendCeilingExceededError`) and the **quota self-throttle** (P0.8: `CallBudget.windowUsage(now)` vs `RUNNER_WINDOW_TOKEN_BUDGET × RUNNER_QUOTA_FRACTION` → `RunnerQuotaError` before spawning, leaving headroom for the human); spawn `claude -p --output-format json --max-turns 1 --model <runner.model>` with the prompt (system+user) on stdin, **tools stripped** (`--disallowedTools Bash,Edit,Write,NotebookEdit,WebFetch,WebSearch,Task`, no `--mcp-config`), `RUNNER_COMPLETION_TIMEOUT_MS` watchdog; return `{ text: envelope.result, usage }`. NO MCP connection (that's agent mode / phase-19).
- **`index.ts`** — barrel + a `Runner` facade wiring binary/health/lanes/spawn/completion over `{ db, settings loader, telemetry, callBudget }`.

## Boot (`index.ts`) — the injections + zombie defense (P0.4/§10.1)
- Build the `Runner`; inject `subscriptionComplete = runner.complete` and `runnerHealthy = runner.isHealthy` **into the phase-16 `ProviderRouter`** (they were left unset). Now a subscribable role routes to subscription iff `runner.enabled ∧ healthy`.
- **`will-quit`:** kill the runner child tree **BEFORE** `mcpServer.stop()` (today it stops MCP first — reorder so a live child isn't stranded) and before `queue.stop()`.
- **Boot:** sweep stale `<userData>/runner/*.mcp.json`; **kill-on-boot** any `runner_runs` row with `is_error IS NULL AND exit_code IS NULL` whose `pid` still resolves to a process whose image matches the resolved runner binary (NEVER kill by pid alone — pids recycle). (`rotateRunnerToken()` on boot already landed in 14b.)
